PHASE 1 Environment Setup
1
Install Python & create virtual environment
Isolate your project dependencies cleanly
▶

First, confirm Python 3.10+ is installed. Then create a clean virtual environment so packages don't conflict with system Python.

# Check Python version first python3 --version # Create virtual environment named 'ragenv' python3 -m venv ragenv # Activate it (Linux/Mac) source ragenv/bin/activate # On Windows use: ragenv\Scripts\activate # You'll see (ragenv) prefix in terminal when active
All subsequent pip installs must be done inside this activated environment. If you close the terminal, re-run the activate command.
2
Install all required libraries
LangChain + ChromaDB + Ollama or OpenAI
▶

Install the core stack. You can use either OpenAI API (paid) or Ollama (free, local LLM) as your language model.

# Core RAG stack pip install langchain langchain-community langchain-core pip install chromadb # vector store pip install sentence-transformers # free local embeddings pip install pypdf unstructured # PDF/document loaders pip install python-dotenv # manage API keys safely pip install fastapi uvicorn # for API later # OPTION A: OpenAI (paid, best quality) pip install langchain-openai openai # OPTION B: Ollama (free, runs locally) pip install langchain-ollama
Save a requirements.txt: pip freeze > requirements.txt so you can recreate the env anytime.
3
Set up Ollama (free local LLM, no API key needed)
Best for development without cost
▶

Ollama runs open-source models like LLaMA3, Mistral, Gemma directly on your machine. No internet needed after download.

# Install Ollama (Linux) curl -fsSL https://ollama.com/install.sh | sh # Mac: download from ollama.com/download # Pull a model (choose one based on your RAM) ollama pull llama3.2 # 2GB RAM, fast ollama pull mistral # 4GB RAM, good quality ollama pull llama3.1:8b # 8GB RAM, best open-source # Start Ollama service (it runs on port 11434) ollama serve # Test it works (new terminal) ollama run llama3.2 "Say hello"
Ollama automatically serves a REST API at http://localhost:11434 — this is what LangChain connects to.

PHASE 2 Data Ingestion Pipeline
5
Write the data ingestion script (ingest.py)
Load → chunk → embed → store in ChromaDB
▶

This script processes your source documents and builds the vector index. Run it once (or re-run when data changes).

# ingest.py — paste this entire file from langchain_community.document_loaders import ( PyPDFLoader, CSVLoader, TextLoader, DirectoryLoader ) from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_community.vectorstores import Chroma import os # 1. Load all documents from the data/ folder loader = DirectoryLoader( "data/", glob="**/*.pdf", # change to *.txt or *.csv loader_cls=PyPDFLoader, show_progress=True ) documents = loader.load() print(f"Loaded {len(documents)} document pages") # 2. Split into chunks (critical for retrieval quality) splitter = RecursiveCharacterTextSplitter( chunk_size=500, # characters per chunk chunk_overlap=50, # overlap prevents lost context separators=["\n\n", "\n", ".", " "] ) chunks = splitter.split_documents(documents) print(f"Created {len(chunks)} chunks") # 3. Create embeddings (free, runs locally) embeddings = HuggingFaceEmbeddings( model_name="sentence-transformers/all-MiniLM-L6-v2" ) # 4. Store in ChromaDB (persists to disk) vectordb = Chroma.from_documents( documents=chunks, embedding=embeddings, persist_directory="./vectorstore" ) vectordb.persist() print("Vector store created successfully!")
# Run the ingestion (place your files in data/ first) python ingest.py
First run downloads the embedding model (~90MB). Subsequent runs are fast. The vectorstore/ folder stores your indexed data permanently.
6
Build the chatbot (chatbot.py)
Query → retrieve chunks → LLM generates answer
▶
# chatbot.py — full RAG chain from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_community.vectorstores import Chroma from langchain_ollama import ChatOllama # free, local # OR: from langchain_openai import ChatOpenAI # paid from langchain.chains import RetrievalQA from langchain.prompts import PromptTemplate # Load the same embeddings used during ingestion embeddings = HuggingFaceEmbeddings( model_name="sentence-transformers/all-MiniLM-L6-v2" ) # Load the persisted vector store vectordb = Chroma( persist_directory="./vectorstore", embedding_function=embeddings ) # Create retriever (top-k similar chunks) retriever = vectordb.as_retriever( search_type="similarity", search_kwargs={"k": 4} # retrieve 4 most relevant chunks ) # Custom prompt — tells LLM to use only your data PROMPT = PromptTemplate( input_variables=["context", "question"], template=""" You are a helpful assistant. Use ONLY the context below to answer. If the answer is not in the context, say "I don't know based on provided data." Context: {context} Question: {question} Answer:""" ) # LLM — choose Ollama (free) or OpenAI (paid) llm = ChatOllama(model="llama3.2", temperature=0) # llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Build the full RAG chain qa_chain = RetrievalQA.from_chain_type( llm=llm, chain_type="stuff", retriever=retriever, chain_type_kwargs={"prompt": PROMPT}, return_source_documents=True ) # Interactive loop to test print("RAG Chatbot ready. Type 'quit' to exit.\n") while True: query = input("You: ") if query.lower() == "quit": break result = qa_chain.invoke({"query": query}) print(f"\nBot: {result['result']}") print("\nSources used:") for doc in result['source_documents']: print(f" - {doc.metadata.get('source','?')} p.{doc.metadata.get('page','?')}") print()
python chatbot.py

PART A Wrap your chatbot as a REST API
1
Create FastAPI server (api.py)
Expose your RAG chatbot as an HTTP endpoint
▶
# api.py — Full FastAPI server from fastapi import FastAPI, HTTPException, Depends from fastapi.middleware.cors import CORSMiddleware from pydantic import BaseModel from typing import Optional import uvicorn, uuid, time # Import your RAG components from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_community.vectorstores import Chroma from langchain_ollama import ChatOllama from langchain.chains import RetrievalQA from langchain.prompts import PromptTemplate app = FastAPI(title="RAG Chatbot API", version="1.0") # CORS — allow frontend to call this API app.add_middleware(CORSMiddleware, allow_origins=["*"], # restrict in production allow_methods=["*"], allow_headers=["*"] ) # Load RAG chain at startup (once, not every request) @app.on_event("startup") async def startup(): global qa_chain embeddings = HuggingFaceEmbeddings( model_name="sentence-transformers/all-MiniLM-L6-v2" ) vectordb = Chroma(persist_directory="./vectorstore", embedding_function=embeddings) retriever = vectordb.as_retriever(search_kwargs={"k": 4}) llm = ChatOllama(model="llama3.2", temperature=0) PROMPT = PromptTemplate( input_variables=["context","question"], template="Context: {context}\n\nQuestion: {question}\nAnswer:" ) qa_chain = RetrievalQA.from_chain_type( llm=llm, retriever=retriever, chain_type_kwargs={"prompt": PROMPT}, return_source_documents=True ) # Request/Response models class QueryRequest(BaseModel): question: str session_id: Optional[str] = None filters: Optional[dict] = None class QueryResponse(BaseModel): answer: str sources: list session_id: str processing_time_ms: int # Main chat endpoint @app.post("/api/chat", response_model=QueryResponse) async def chat(req: QueryRequest): if not req.question.strip(): raise HTTPException(400, "Question cannot be empty") start = time.time() result = qa_chain.invoke({"query": req.question}) elapsed = int((time.time() - start) * 1000) sources = [{ "file": doc.metadata.get("source","?"), "page": doc.metadata.get("page","?"), "content": doc.page_content[:200] } for doc in result["source_documents"]] return QueryResponse( answer=result["result"], sources=sources, session_id=req.session_id or str(uuid.uuid4()), processing_time_ms=elapsed ) @app.get("/health") async def health(): return {"status": "ok", "model": "llama3.2"} if __name__ == "__main__": uvicorn.run(app, host="0.0.0.0", port=8000, reload=True)
# Run the API server python api.py # Or with auto-reload (development mode) uvicorn api:app --reload --port 8000
FastAPI auto-generates interactive docs at http://localhost:8000/docs — you can test every endpoint visually without writing any code.

Full Python integration with data.gov.in
Fetch, filter, paginate — micro-step code
▶
# datagov_client.py import requests, json, os from dotenv import load_dotenv load_dotenv() API_KEY = os.getenv("DATAGOV_API_KEY") BASE_URL = "https://api.data.gov.in/resource" def fetch_data(resource_id, filters=None, limit=100, offset=0): """ resource_id: the dataset's unique ID from data.gov.in filters: dict like {"state": "Gujarat", "year": "2021"} """ params = { "api-key": API_KEY, "format": "json", "limit": limit, "offset": offset } # Add field filters (key=value pairs) if filters: for key, value in filters.items(): params[f"filters[{key}]"] = value url = f"{BASE_URL}/{resource_id}" response = requests.get(url, params=params, timeout=30) response.raise_for_status() return response.json() def fetch_all_pages(resource_id, filters=None, page_size=500): """Paginate through all results""" all_records = [] offset = 0 while True: data = fetch_data(resource_id, filters, page_size, offset) records = data.get("records", []) total = int(data.get("total", 0)) all_records.extend(records) offset += len(records) print(f"Fetched {offset}/{total} records...") if offset >= total or not records: break return all_records # Example usage if __name__ == "__main__": RESOURCE_ID = "YOUR_RESOURCE_ID_HERE" # Basic fetch (first 10 records) data = fetch_data(RESOURCE_ID, limit=10) print("Total records:", data.get("total")) print("Fields:", data.get("field", [])) print("Sample:", json.dumps(data["records"][0], indent=2)) # Filtered fetch gujarat_data = fetch_data(RESOURCE_ID, filters={"state": "Gujarat"}, limit=50) # Fetch all and save to JSON all_data = fetch_all_pages(RESOURCE_ID) with open("data/govdata.json", "w") as f: json.dump(all_data, f, indent=2) print(f"Saved {len(all_data)} records")

7
Ingest government API data into your RAG chatbot
Bridge govt data with your chatbot pipeline
▶

This is the key step — pull data from any govt API and feed it into your RAG vector store automatically.

# govt_rag_ingest.py — Pull live govt data into RAG import requests, json from langchain.schema import Document from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_community.vectorstores import Chroma API_KEY = "YOUR_DATAGOV_KEY" def fetch_and_ingest(resource_id, description): """Fetch from data.gov.in and add to vector store""" # Step 1: Fetch data url = f"https://api.data.gov.in/resource/{resource_id}" resp = requests.get(url, params={ "api-key": API_KEY, "format": "json", "limit": 500 }) data = resp.json() # Step 2: Convert records to text Documents docs = [] for record in data.get("records", []): # Convert each row to readable text text = f"{description}:\n" text += "\n".join([f"{k}: {v}" for k, v in record.items()]) docs.append(Document( page_content=text, metadata={ "source": f"data.gov.in/{resource_id}", "type": "government_data", "description": description } )) # Step 3: Chunk and embed splitter = RecursiveCharacterTextSplitter( chunk_size=500, chunk_overlap=50) chunks = splitter.split_documents(docs) embeddings = HuggingFaceEmbeddings( model_name="sentence-transformers/all-MiniLM-L6-v2") vectordb = Chroma(persist_directory="./vectorstore", embedding_function=embeddings) vectordb.add_documents(chunks) vectordb.persist() print(f"Added {len(chunks)} chunks from {description}") # Example: Add population + literacy data fetch_and_ingest( "6176ee09-3d56-4a3b-8115-21841daa7f85", "India State Population Census 2021" ) fetch_and_ingest( "YOUR_LITERACY_RESOURCE_ID", "India District Literacy Rates" )
After running this, your chatbot can answer questions like "What is Gujarat's literacy rate?" or "Which state has highest population growth?" using live government data.

PRODUCTION CHECKLIST
5
Before going to production — complete checklist
Security, performance, monitoring
▶

Security

  • API keys in .env, not in code
  • Rate limiting per IP/key
  • CORS restricted to your domain
  • HTTPS (use nginx + SSL)
  • Input validation & max length

Performance

  • Load RAG chain once at startup
  • Add response caching (Redis)
  • Async endpoints (FastAPI async)
  • Optimise chunk size (test 300–800)
  • Use GPU if available (faster LLM)

Monitoring
